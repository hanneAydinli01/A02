PART 1: Directions on Using Webstorm.


STEP 1: Install Git

1. Go to the official Git website:
https://git-scm.com/

2. Download Git for your operating system (Mac or Windows).

3. Install Git using default settings.

4. Open Terminal and type:

git --version

If Git is installed correctly, you will see the version number.


STEP 2: Install WebStorm

1. Go to the official JetBrains website:
https://www.jetbrains.com/webstorm/

2. Download WebStorm.

3. Install the software.

4. Open WebStorm and sign in if required.


STEP 3: Create a Repository in GitHub

1. Go to https://github.com
2. Click “New Repository”.
3. Name it A02.
4. Make it Public.
5. Add a README file.
6. Click Create Repository.


STEP 4: Clone Repository in WebStorm

1. Copy the repository URL.
2. In WebStorm click:
   File → New → Project from Version Control.
3. Paste the repository URL.
4. Click Clone.

Now the repository is connected to your computer.


STEP 5: Make Changes and Commit

1. Edit the README.md file.
2. Go to the Git tab.
3. Click Commit.
4. Write a clear commit message.
5. Click Commit and Push.


STEP 6: Push Changes

Push uploads your changes to GitHub.


STEP 7: Pull Changes

Pull downloads updates from GitHub to your local computer.

PART 2: Glossary to include these terms in a bulleted list.


 **Branch** – A separate version of a project used to develop features without affecting the main project.

 **Clone** – To copy a repository from GitHub to your computer.

 **Commit** – A saved change in a Git project with a message explaining what was changed.

 **Fetch** – A command that checks for updates from the remote repository without changing local files.

 **GIT** – A version control system used to track changes in source code.

 **Github** – A cloud-based platform that hosts Git repositories online.

 **Merge** – Combining changes from one branch into another.

 **Merge Conflict** – When Git cannot automatically combine changes and needs manual correction.

 **Push** – Uploading local commits to GitHub.

 **Pull** – Downloading changes from GitHub to your local repository.

 **Remote** – The online version of a repository hosted on GitHub.

 **Repository** – A storage location for project files and version history.
